Documentación de los 'typedefs'

◆ ComponentIndex

using OSK::ECS::ComponentIndex = typedef TSize

Identificador único para un componente.

Un componente es una estructura que almacena datos específicos para uno o varios sistemas. Los componentes no ejecutan lógica alguna, y sólo son contenedores de información relacionada con un GameObject.

Atención
Para que una clase pueda ser usada como componente, debe tener implementado OSK_COMPONENT(className).
Este debe ser un identificador único: no pueden haber dos tipos de componentes con el mismo className.
Nota
Cada GameObject sólo puede tener un componente de cada tipo.

◆ Signature

using OSK::ECS::Signature = typedef ConstexprBitSet<MAX_COMPONENT_TYPES>

Un signature permite saber qué componentes contiene un GameObject. Es un bitmap, donde el bit en la posición 'X' indicará si el GameObject tiene un componente del tipo (ComponentType == X).

Los sistemas también tienen su propio signature, que indica qué componentes necesita tener un GameObject para ser procesado por ese sistema.
