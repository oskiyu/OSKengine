◆ GpuSharedMemoryType

enum class OSK::GRAPHICS::GpuSharedMemoryType
strong

Define qué dispositivo tiene acceso a la memoria.

Puede definir una memoria exclusiva para la GPU (mayor rendimiento) o una memoria accesible tanto desde la CPU como la GPU.

Valores de enumeraciones
GPU_AND_CPU 

Compartido. Útil para recursos que van a ser sobreescritos a menudo, o recursos de un solo uso a los que vamos a escribir información directamente, como buffers intermedios.

GPU_ONLY 

Exclusivo para la GPU. Mejor rendimiento.

Atención
No se puede escribir directamente desde la CPU: se debe crear un buffer intermedio al que tengamos acceso (GpuSharedMemoryType::GPU_AND_CPU), y después copiar sus contenidos usando la cola de comandos.

◆ PresentMode

enum class OSK::GRAPHICS::PresentMode
strong

Define el modo de sincronización vertical.

define a qué ritmo se actualiza el fotograma presentado en el monitor.

Valores de enumeraciones
VSYNC_OFF 

No se espera a que la imagen actual termine de presentarse para mostrar la nueva imagen.

Produce tearing.

VSYNC_ON 

Espera a que la imagen actual termine de presentarse para mostrar la nueva imagen.

No produce tearing.

VSYNC_ON_TRIPLE_BUFFER 

Espera a que la imagen actual termine de presentarse para mostrar la nueva imagen.

Se sigue procesando nuevas imágenes aunque todavía no se puedan presentar, permitiendo presentar una imagen más reciente si lo comparas con PresentMode::VSYNC_ON.

No produce tearing.
