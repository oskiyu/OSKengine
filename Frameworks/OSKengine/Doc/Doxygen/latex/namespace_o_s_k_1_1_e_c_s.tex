Identificador único para un componente. 

Un componente es una estructura que almacena datos específicos para uno o varios sistemas. Los componentes no ejecutan lógica alguna, y sólo son contenedores de información relacionada con un Game\+Object.

\begin{DoxyWarning}{Atención}
Para que una clase pueda ser usada como componente, debe tener implementado \mbox{\hyperlink{_component_8h_a400adac69bcf856191172f22f63f066c}{OSK\+\_\+\+COMPONENT(class\+Name)}}. 

Este debe ser un identificador único\+: no pueden haber dos tipos de componentes con el mismo class\+Name.
\end{DoxyWarning}
\begin{DoxyNote}{Nota}
Cada Game\+Object sólo puede tener un componente de cada tipo. 
\end{DoxyNote}

Un signature permite saber qué componentes contiene un Game\+Object. Es un bitmap, donde el bit en la posición \textquotesingle{}X\textquotesingle{} indicará si el Game\+Object tiene un componente del tipo (Component\+Type == X). 

Los sistemas también tienen su propio signature, que indica qué componentes necesita tener un Game\+Object para ser procesado por ese sistema.
