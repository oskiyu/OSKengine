\doxysubsection{Descripción detallada}
Entity-\/\+Component-\/\+System es un paradigma de programación en el cual los objetos se dividen en 3 partes diferenciables\+: 


\begin{DoxyItemize}
\item Game\+Object\+:
\begin{DoxyItemize}
\item Representa un objeto en concreto.
\item Es simplemente un identificador único.
\end{DoxyItemize}
\item Componente\+:
\begin{DoxyItemize}
\item Estructura que almacena datos específicos de un tipo dado de un Game\+Object.
\item Los componentes no ejecutan lógica alguna, y sólo son contenedores de información relacionada con un Game\+Object.
\end{DoxyItemize}
\item Sistemas\+:
\begin{DoxyItemize}
\item Sencarga de ejecutar lógica con los datos de los componentes.
\end{DoxyItemize}
\end{DoxyItemize}

Esta clase es la encargada de manejar este sistema, proporcionando una capa de abstracción sobre la implementación del sistema.

\begin{DoxyNote}{Nota}
Toda operación que el juego quiera hacer sobre el \mbox{\hyperlink{namespace_o_s_k_1_1_e_c_s}{ECS}} debe hacerse a través de esta clase. 

Esta clase es dueña de los sistemas. 
\end{DoxyNote}
